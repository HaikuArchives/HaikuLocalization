Индексы

Содержание     Предыдущий: Атрибуты     Следующий: Запросы

Атрибуты и Запросы - ключевы возможности Haiku. В то время как атрибуты полезны как таковые, для отображения дополнительной информации о файле, при использовании запросов по ним, они лолжны быть индексированы. Индексирование помещает атрибуты в специальную таблицу поиска, которая облегчаат и ускоряет запросы.
Индексы являются частью файловой системы и хранятся отдельно для каждого тома/раздела.

Команды индексирования в Terminal

Вот некоторые команды управления индексами:

  • lsindex - Отображает индексированные атрибуты текущего тома/раздела.

  • mkindex - Добавляет атрибуты в индексы тома/раздела. 
    Использование: mkindex [options] <attribute> 
Создаёт новый индекс для указанного атрибута. 
 
  -d, --volume=PATH     путь к тому, на котором добавляются индексы
                        по-умолчанию - текущий том. 
  -t, --type=TYPE       тип атрибута, который необходимо проиндексировать.  Один из "int", 
                        "llong", "string", "float", or "double". 
                        По-умолчанию -  "string". 
      --copy-from       путь к тому с которого индексы должны быть скопированы. 
  -v, --verbose         выводить иинформацию о создаваемых индексах

    Заметка: Только новые файлы с этими атрибутами будут автоматически проиндексированы!
    Существующие фацлы должны быть добавлены вручную копировнаием и последующим удалением оригиналов. Альтернативно можно использовать данный инструмент reindex.

  • rmindex - Удаляет атрибуты из индекса тома/раздела. 
    Использование: rmindex [OPTION]... INDEX_NAME 
 
Удаляет индекс с именем INDEX_NAME с раздела диска. После того, как эта операция
будет проделана, становится невозможным использовать систему запросов для поиска
файлов с атрибутом INDEX_NAME. 
 
  -d, --volume=PATH     путь к тому, с которого производится удаление
                         индекса 
  -h, --help            отображение этой справки и выход 
  -p, --pattern         образец INDEX_NAME 
  -v, --verbose         выводить иинформацию об удаляемых индексах 
 
INDEX_NAME - имя атрибута файла. 
 
Если том не указан, используется том текущей директории.
